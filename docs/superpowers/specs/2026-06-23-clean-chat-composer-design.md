# 干净聊天框与组合模型菜单设计

## 目标

把 AE 插件的聊天页改成接近 Codex 的一体式对话界面：移除持续占位的 Token、上下文和模型工具栏，将常用控制集中到大输入框底部，同时保留上下文保护、模式安全与 AE 动作确认流程。

## 页面结构

聊天页不再显示顶部 Token 进度条、独立上下文档案行、`AI 对话工作区`工具栏及顶部模型选择框。消息区作为主要滚动区域，底部固定一个大圆角输入框。

无消息时，消息区正中央仅显示“你好”和“今天想制作什么？”，不显示机器人图标或长说明。用户发送第一条消息后，空状态完全消失。AE 工程连接状态只在“操作 AE”模式下以弱化小字显示，不再占用永久工具栏。

输入框上半部分用于多行提示词，下方控制栏从左到右排列：`+` 菜单、对话模式、组合模型按钮、可选的上下文数量标记；发送按钮位于右侧。窄面板时控制栏允许模型名称省略显示，但发送按钮始终可见。

## 菜单与状态

`+` 菜单包含“上下文档案”入口。用户可在子面板中多选 Markdown 上下文档案；选中后输入框底栏显示“上下文 N”，未选中时不显示标记。原有档案编辑能力保留，但不再常驻聊天页顶部。

“普通对话 / 操作 AE”使用独立下拉按钮，继续写入 `AppState.chatMode`，关闭面板或重启 AE 后保持，直到用户手动切换。

模型选择合并为一个按钮。按钮只显示当前模型名；展开后按 API 供应商分组列出所有已保存且支持聊天的模型。点击模型时同时更新聊天能力的供应商与模型选择。没有可用模型时按钮显示“选择模型”，发送按钮不可用，并显示简短配置提示。

## Token 与安全

聊天页不显示 Token 数值、百分比或常驻进度条。现有 Token 估算、供应商 usage 记录、上下文上限和 80% 警告逻辑继续在后台运行。

只有上下文达到警告或阻止阈值时，输入框上方才出现紧凑提示，提供“压缩并续聊”操作；未达到阈值时不占用空间。达到阻止阈值后继续禁止直接发送，不进行静默裁剪。

普通对话模式继续拒绝任何 AE 动作计划；操作 AE 模式的信息不足回复自然追问，信息完整后生成动作预览。所有动作必须经过现有确认流程，删除图层或关键帧继续二次确认。内部 JSON 信封不显示在聊天历史或请求等待状态中。

## 组件边界

新增组合聊天模型选择组件，负责按供应商分组、展示当前模型和更新 `activeSelections.chat`。组件依赖现有模型选择工具函数，不修改 API 档案结构。

聊天页负责页面布局、空状态条件、`+` 菜单、上下文选择和 Token 临界提示。上下文编辑器、消息渲染、动作预览和归档逻辑继续复用现有实现，不进行无关重构。

## 验证

- 单元测试覆盖组合模型列表分组、供应商与模型同步切换、无模型状态。
- 界面测试覆盖 Token 条和旧工具栏不存在、空状态居中、首条消息后消失、模式菜单持久化、上下文数量标记及组合模型选择。
- 临界上下文测试覆盖平时不显示提示、警告时显示压缩入口、阻止时禁止发送。
- 完整运行 Vitest、TypeScript、Vite 构建和两条 Playwright 预览测试。
- 验证通过后重新安装到当前用户的 AE CEP 扩展目录，并推送 GitHub `master`。
