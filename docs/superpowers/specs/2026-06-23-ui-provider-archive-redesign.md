# AE AI Assistant 交互、模型供应商与归档改版设计

## 目标

解决按钮缺少点击反馈、API 档案编辑体验不明确、供应商配置门槛高、聊天区域层次不足、模型无法按任务切换，以及归档对话固定保存在系统盘的问题。

## 交互反馈

- 所有按钮提供统一的 `hover`、`active` 和 `disabled` 状态。
- 按下时缩放至 0.96，松开后恢复；主要按钮同时改变边框或背景色。
- 保存、连接测试、模型同步、生成、导入和归档操作显示进行中状态及明确的成功/失败反馈。
- API 档案保存结果显示在表单附近，不能只依赖面板底部状态栏。

## API 档案与供应商预设

供应商预设包括 OpenAI、DeepSeek、Moonshot/Kimi、阿里云百炼/通义千问、智谱 GLM、小米 MiMo、火山引擎豆包和自定义 OpenAI-compatible。

选择预设后自动填写供应商名称、Base URL、模型端点、聊天端点和已知能力。用户主要填写 API Key，再通过模型接口获取可用模型。预设不硬编码可能快速过期的模型列表。

API 档案使用显式草稿编辑：

- 点击已保存档案进入编辑状态。
- 表单始终允许修改；“放弃修改”恢复已保存值。
- 保存时使用稳定 ID 更新原档案，不创建重复项。
- API Key 留空表示保留原密钥；填写新值才替换 DPAPI 密文。
- 保存成功后仍停留在当前档案，并显示“已保存”状态。
- 每个档案缓存最近一次同步的模型 ID、上下文长度和更新时间。

## 模型选择

应用状态分别保存 chat、image、video 的活动供应商档案和模型。

- 聊天页顶部提供供应商和模型两个选择器。
- 图片与视频生成页分别提供供应商和模型选择器。
- 选择器只显示具备对应能力的已保存档案。
- 模型选择优先使用供应商同步结果；没有同步结果时允许手动输入。
- 每次请求使用当前页面选择的模型覆盖档案默认模型，不修改其他能力的选择。

## 聊天页布局

- 将对话历史、动作计划和输入框放进带标题栏的独立对话容器。
- 标题栏显示当前供应商、模型及 AE 工程连接状态。
- 上下文 Token 状态缩小为单行紧凑条，显示百分比和 `已用/上限`。
- MD 上下文档案保留为可横向滚动的标签区。
- 对话输入框仍保持在容器底部，动作计划显示在历史与输入框之间。

## 外部对话归档

- 历史页增加“对话归档目录”和“选择目录”操作。
- 目录通过 CEP 文件夹选择器选择，可位于任意本地磁盘。
- 归档会话时将完整内容写为 UTF-8 Markdown，文件名使用安全化标题、日期和会话 ID。
- Markdown 包含标题、创建时间、使用的上下文档案、Token 用量和逐条消息。
- 归档成功后，系统盘状态文件仅保留会话标题、时间、交接摘要和外部文件路径，移除完整历史消息。
- 未设置或目录不可写时阻止归档，并要求重新选择目录；不得静默回退到 C 盘。
- API 密钥继续使用 Windows DPAPI 保存，不随归档文件移动。

## 数据结构与迁移

- `ApiProfile` 增加供应商类型、模型缓存和最后同步时间。
- `AppState` 增加三个能力的活动模型选择和归档目录。
- `Conversation` 增加归档文件路径与交接摘要。
- 加载旧状态时补齐新增字段，保留已有档案、对话、模板和任务。
- 旧会话只有在用户主动归档时才移出系统盘。

## 错误处理

- 模型接口不返回上下文长度时保留模型，并标记长度未知。
- 供应商预设端点请求失败时允许用户切换到高级配置修正。
- 当前模型在重新同步后消失时保留原值并显示“模型可能已不可用”。
- 外部归档写入采用临时文件加原子重命名，失败时不清空内存中的会话。
- API 档案保存与 DPAPI 密钥更新任一步失败时显示错误，档案草稿保持可编辑。

## 测试与验收

- 单元测试覆盖供应商预设、旧状态迁移、活动模型选择、档案稳定 ID 更新和归档 Markdown。
- 回归测试验证档案保存、离开页面、重新进入、修改并再次保存。
- 交互测试验证按钮按下样式、聊天/生成模型切换、紧凑 Token 条和独立对话容器。
- 文件测试验证自定义目录写入、原子保存、非法目录失败及系统盘状态中不再保留完整归档消息。
- 构建后重新安装 CEP 扩展，并在 430×820 面板尺寸下完成视觉 QA。

## 不在本次范围

- 不改变现有 `ae-actions/v1` 动作协议。
- 不增加云端账号、同步或计费系统。
- 不移动 API 密钥到外部归档目录。
- 不在预设中维护固定模型价格或长期硬编码模型列表。
