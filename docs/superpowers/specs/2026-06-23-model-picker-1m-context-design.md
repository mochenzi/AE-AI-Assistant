# 模型下拉选择与 1M 上下文声明设计

## 问题

AE CEP 使用的旧版 Chromium 会绘制 `datalist` 的倒三角，但点击后不一定显示候选项。模型同步成功后，用户因此无法可靠切换模型。同时，部分供应商不返回上下文长度，需要允许用户明确声明某个模型支持 1M tokens。

## 方案

使用标准 `<select>` 取代聊天、图片、视频和 API 档案编辑页中的模型 `datalist`。缓存模型存在时显示真实下拉列表；列表末尾提供“手动输入模型…”，选择后切换为文本输入，并可返回模型列表。

在 API 档案的 chat 模型配置旁增加“声明支持 1M”复选框。勾选后只把当前 chat 模型的缓存上下文长度设为 `1,000,000`，取消后恢复供应商同步值；供应商未提供长度时恢复为未设置。该值随 API 档案持久化，并驱动主聊天页上下文进度条和发送限制。

模型切换时，复选框读取目标模型自己的上下文长度，不把 1M 声明复制到其他模型。图片和视频模型不显示上下文声明。

## 备选方案

- 保留 `datalist` 并修 CSS：无法解决 CEP Chromium 的原生控件兼容问题，不采用。
- 仅使用 `<select>`：无法输入模型列表中尚未同步的新模型，不采用。
- 自制浮层下拉框：交互成本和焦点兼容风险高于标准控件，首版不采用。

## 数据与迁移

在 `ApiProfile.cachedModels[]` 增加可选字段 `declaredContextWindow`。供应商同步值继续保存在 `contextWindow`，用户声明保存在 `declaredContextWindow`；有效长度优先使用声明值。旧档案无需迁移。同步模型时按模型 ID 保留声明字段，取消勾选后自然恢复供应商同步值。

## 验证

- 单元测试：标准模型选择、手动输入、按模型保存/取消 1M 声明、模型同步保留声明。
- 浏览器测试：同步模型后标准下拉框可打开并切换；选择“手动输入”后可填写自定义模型。
- 完整检查：TypeScript、Vitest、Vite 构建、CEP 安装。
