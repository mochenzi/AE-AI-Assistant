# 普通对话与 AE 操作模式设计

## 问题

当前聊天系统提示词要求模型只能返回 `ae-actions/v1` JSON，并且请求默认启用 JSON Object。普通问题因此也会被模型解释成 AE 动作，用户能看到内部 JSON，而不是自然语言回复。

## 交互

输入框左侧增加 `+` 模式菜单，包含“普通对话”和“操作 AE”。当前模式以清晰的状态标签显示，并持续保持到用户手动切换；模式写入本地状态，关闭面板或重启 AE 后仍保留。

普通对话模式禁止生成 AE 动作，只显示自然语言。操作 AE 模式允许 AI 在信息不足时继续自然追问；信息完整时生成动作预览。任何动作都不能自动执行，必须经过现有确认流程，删除类危险动作继续二次确认。

## 响应协议

模型内部统一返回 JSON 信封，界面不显示信封本身：

```json
{"kind":"chat","message":"自然语言回答或追问"}
```

```json
{"kind":"ae_action","plan":{"version":"ae-actions/v1","summary":"…","risk":"low","projectRevision":"…","actions":[]}}
```

普通模式的系统提示词只允许 `kind: chat`。操作模式允许两种 kind，并明确要求缺少目标、数值或范围时返回追问。解析失败时把原始文本作为普通回复显示，不产生动作计划。

## 状态与安全

在 `AppState` 增加 `chatMode: "chat" | "ae"`，旧状态迁移默认使用 `chat`。发送请求时读取当前模式；操作计划仍经过 Schema 校验、工程 revision 校验、一次确认或危险操作二次确认。

历史消息只保存用户实际看到的自然语言，动作信封只保存经过解析后的摘要和预览状态，不把内部 JSON 当作助手消息展示。

## 验证

- 单元测试覆盖信封解析、普通文本回退、无效动作降级、状态迁移默认值。
- 界面测试覆盖模式菜单、模式持久保持、普通问答不出现动作 JSON、操作模式追问和动作预览。
- 完整运行 TypeScript、Vitest、Vite 构建、Playwright 与 CEP 安装。
