# Codex 式会话、Markdown 与活动合成上下文设计

## 目标

把 AE 插件的聊天区升级为适合长期工作的独立会话系统，同时解决“操作 AE”模式下模型偶发返回空内容的问题。用户可以按 AE 工程管理多个对话，在创建对话时一次性载入 Markdown，并通过输入框的 `+` 菜单按需附加最新的活动合成结构。

首版继续限定 Windows、AE 25/26 和本机自用。不上传截图、媒体或 `.aep` 文件，不扩大 AE 动作白名单，也不取消动作预览、目标版本校验、危险操作二次确认或撤销能力。

## 交互设计

聊天页增加可折叠的会话抽屉。抽屉包含“新对话”、本地搜索、当前工程名称和该工程的会话列表。会话按 `.aep` 的规范化绝对路径分组；未保存工程进入“未保存工程”分组。工程首次保存后，该临时分组中的会话迁移到已保存工程分组。窄面板自动收起抽屉，只保留展开按钮。

点击“新对话”时显示初始化弹窗。用户先选择“不使用 Markdown”或选择一个或多个本地 `.md` 文件，再创建会话。Markdown 仅在创建时读取一次，之后原文件发生变化不会影响既有会话。每个会话保存自己的 Markdown 快照，切换会话不会混用上下文。新对话默认不继承上一会话的 Markdown。

输入框的 `+` 菜单保留“上下文档案”能力，并增加“当前合成内容”开关。开启后，输入框显示“合成名 · 图层数 · 发送时刷新”的可移除标签；状态保持到用户手动关闭。每次发送前重新读取当前活动合成。如果没有活动合成，停止发送并提示先打开一个合成。

每条会话相互独立，保存自己的消息、模型选择、Markdown 快照、合成上下文开关和 Token 统计。会话标题默认取第一条用户消息的精简文本，用户可以重命名。切换或恢复会话绝不自动执行 AE 动作。

## 本地存储

首次创建新对话时，如果尚未设置“对话数据目录”，要求用户选择一个可写目录。目录可在设置中修改。完整消息和 Markdown 快照不写入 C 盘的应用状态文件；应用状态只保存数据目录路径、当前会话 ID 和用于快速显示的最小索引。

数据目录按工程建立子目录，每个会话独立保存。会话写入采用临时文件加原子重命名。索引损坏或缺失时可以从会话文件重建。数据目录丢失或不可写时，保留当前内存内容，停止持久化并提示重新选择目录，不静默切换回系统盘。

本地搜索仅查询会话标题和消息文本的本地索引，不调用模型或联网。日志不得记录 Markdown 正文、完整会话正文、API Key、Authorization 或签名 URL。

## 活动合成快照

新增独立的 ExtendScript 读取入口，用于生成只读的活动合成快照，不复用 AE 动作执行入口。快照包含：

- 工程名、工程路径、工程 revision。
- 合成 ID、名称、尺寸、像素宽高比、时长、帧率、工作区和当前时间。
- 全部图层的索引、名称、类型、顺序、开关、锁定、父子关系、开始/入点/出点。
- 文本图层的源文本和关键文本样式摘要。
- 常用变换属性、已修改属性、表达式摘要。
- 效果名称、启用状态和可安全读取的已修改参数。
- 关键帧数量、时间、插值摘要和可序列化的值。

读取过程必须逐属性容错；某个第三方效果或属性读取失败时记录该字段不可用，不能让整个快照失败。快照不包含画面像素、音视频数据、素材二进制或工程文件内容。

为防止 AE 卡顿和 Token 暴涨，序列化器实施明确上限。优先保留选中图层，再按图层顺序保留其他图层；长文本、表达式、效果参数和关键帧列表分别限量。超限时输出结构化的 `truncated` 信息，说明哪些图层或属性未完整发送。快照占用不得超过模型有效上下文预算的 25%；达到预算后停止继续展开，禁止无提示裁剪。

## 请求组装与安全

发送顺序为：系统安全提示、会话 Markdown 快照、最新活动合成快照、会话历史、当前用户消息。Markdown 和 AE 快照均标记为不可信参考资料，不能覆盖系统提示、动作协议、动作白名单、确认步骤或危险操作规则。

只有用户当前选择的 API 档案接收上述结构化文本。插件不把工程、媒体、截图或本地文件上传到其他服务。界面在发送前显示该会话已附加的 Markdown 文件名，以及活动合成是否开启。

AI 返回动作计划后继续使用快照中的工程 revision。执行前重新读取 revision；工程、活动合成、图层数量或选择状态发生变化时将计划标记为过期，要求重新生成。

## 空响应恢复

部分供应商在 `response_format: {"type":"json_object"}` 下可能返回成功状态但没有最终 `content`。聊天客户端必须先完整解析本次 SSE 事件，再决定是否完成：

1. 首次请求按 API 档案配置发送。
2. 如果结构化模式为 `json_object`、HTTP 成功但没有任何最终内容，则自动重试一次，并仅移除 `response_format`；严格的 JSON 动作信封系统提示保持不变。
3. `reasoning_content` 只用于判断和诊断，不作为最终回复展示，也不作为动作 JSON 执行。
4. 第二次仍为空时抛出明确错误，提示用户重试或把档案改为“纯提示词”。不写入“AI 没有返回内容”这样的伪造助手消息。
5. 只有首次空响应时才产生第二次请求，禁止循环重试；如果首次已有内容，不得重复请求或重复记录 usage。

## 错误处理

- Markdown 创建时读取失败：显示具体文件名和原因，允许移除失败文件；未解决前不创建不完整会话。
- 没有活动合成：当“当前合成内容”开启时阻止发送，并显示可操作提示。
- 合成属性读取失败：保留其余快照，并在 `unavailable` 列表中说明失败字段。
- 数据目录不可写：保留内存状态并要求重新选择目录，不丢弃消息。
- 会话文件损坏：隔离损坏文件，显示恢复提示，并从其余文件重建索引。
- API 空响应：最多自动重试一次；再次失败后保持用户消息可重新发送。
- 动作目标过期：禁止执行，要求基于最新合成快照重新生成。

## 代码边界

- CEP UI：会话抽屉、新对话弹窗、Markdown 文件选择、附件标签、搜索与错误提示。
- Node 服务层：会话目录、原子存储、本地索引、Markdown 一次性读取、快照 Token 限量和 API 空响应重试。
- ExtendScript：只读活动合成快照和既有 AE 动作执行器，两者保持独立入口。
- 共享协议：会话元数据、Markdown 快照、活动合成快照、截断信息和版本化字段。

现有聊天页不承担文件系统细节，ExtendScript 不进行网络请求或本地会话存储，API 客户端不直接操作 React 状态。

## 测试与验收

单元测试覆盖会话工程键、标题生成、索引重建、原子存储、Markdown 一次性读取、会话隔离、快照序列化和截断、日志脱敏，以及空响应一次重试。

模拟 API 覆盖：首次 JSON 空响应后纯提示词成功、首次直接成功、两次空响应、仅含 `reasoning_content`、401、429、超时和异常 SSE。断言重试请求不带 `response_format`，且 usage 不重复累计。

固定 AE 工程覆盖：无活动合成、普通/文本/形状/预合成/摄像机/灯光图层、父子关系、表达式、效果、关键帧、第三方属性读取异常、超大合成截断和目标过期。

界面测试覆盖：新对话、多个 Markdown、工程分组、切换、重命名、搜索、抽屉自动收起、活动合成开关保持、附件标签和错误恢复。

最终在 AE 25/26 验收闭环：新对话 → 选择 Markdown → 开启当前合成 → 提问 → 收到正常回复或动作预览 → 确认执行 → 撤销。重启 AE 后会话列表、当前会话及附件状态能够恢复，且不会自动执行任何动作。
